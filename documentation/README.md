# documentation/

This folder contains the full suite of project documents generated by Claude Code.

Each subfolder is a documentation layer, numbered in dependency order — governance and requirements first, then design, architecture, and operations last.

| Layer | Folder | What it contains |
|---|---|---|
| 00 | `00-governance/` | Project charter, RACI matrix, risk register, change log, definition of done |
| 01 | `01-requirements/` | Glossary, stakeholder map, BRD, personas, user journeys, PRD, NFRs, acceptance criteria |
| 02 | `02-design/` | Data model, flow specs, sequence diagrams, state machines, API design, DB schema, test strategy |
| 03 | `03-data/` | Data dictionary, data flow diagram, seed data strategy |
| 04 | `04-architecture/` | Architecture docs, infrastructure, security, observability, CI/CD flows, ADRs |
| 05 | `05-developer-experience/` | Local setup, coding standards, git workflow, PR guide, developer FAQ |
| 06 | `06-operations/` | Release plan, runbook, incident response, feature flags, hotfix and rollback flows |
| 07 | `07-frontend-design/` | Lo-fi wireframes, hi-fi designs, design tokens, component specs |

> These files are placeholders showing the output structure. They are overwritten with real content when you run the workflow and say `generate docs`.
